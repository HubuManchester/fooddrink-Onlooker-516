## 1. UI/UX Design & Accessibility (30%) / 界面设计与无障碍 (≈ 2 min)

**Show**: Tap through 4 tabs → Toggle system Dark Mode and return to app → Open Colors.xaml in VS → Open RandomPage.xaml, point to SemanticProperties → Point out button sizes on screen

**CN**: App 有四个主要页面：Discover 随机推荐、Compass 美食指南针、Album 美食相册、Favourites 我的收藏。底部 TabBar 导航清晰直观。

所有页面都使用了 MAUI 的 AppThemeBinding 来实现深色模式自动适配。我在 Colors.xaml 里定义了 18 个自定义主题色变量，亮色和暗色各一套。当系统切换到深色模式时，所有页面自动切换配色。这符合 WCAG 1.4.3 颜色对比度原则。

所有交互元素都添加了 SemanticProperties.Description 和 Hint，方便屏幕阅读器识别。按钮最小尺寸 44x44 像素，符合 WCAG 2.5.5 目标尺寸标准。每个输入框都有清晰的 Placeholder 提示文字。

**EN**: The app has four main pages: Discover for random recommendations, Compass for direction-based picks, Album for food photos, and Favourites for saved dishes. The bottom TabBar provides clear, intuitive navigation.

All pages use MAUI's AppThemeBinding for automatic dark mode adaptation. I defined 18 custom theme colour variables in Colors.xaml — one set for light, one for dark. When the system switches to dark mode, every page adapts seamlessly. This aligns with WCAG 1.4.3 contrast requirements.

All interactive elements include SemanticProperties.Description and Hint for screen reader support. Buttons maintain a minimum 44x44 pixel touch target, meeting WCAG 2.5.5 target size guidelines. Every input field has a clear placeholder hint.

## 2. Mobile Hardware (20%) / 移动硬件功能 (≈ 3 min)

### 2.1 Accelerometer — Shake / 加速度计 (≈ 40s)

**Show**: Shake phone → Card animates + new food appears + sound plays

**CN**: 在 Discover 页面，摇晃手机会触发 Accelerometer.ShakeDetected 事件。我加了 800 毫秒的防抖机制避免重复触发。每次摇晃都会随机换一道菜，同时播放签筒摇晃音效和卡片弹跳动画。

**EN**: On the Discover page, shaking the phone triggers the Accelerometer.ShakeDetected event. I added an 800-millisecond debounce to prevent duplicate triggers. Each shake randomly picks a new dish, accompanied by a custom rattle sound effect and card bounce animation.

### 2.2 Compass / Magnetometer / 指南针 (≈ 50s)

**Show**: Switch to Compass tab → Rotate phone slowly → Compass spins + recommendation changes

**CN**: 指南针页面使用 Compass API 读取磁力计数据。我实现了指数平滑滤波来减少传感器噪声——每次新读数只采纳 12%，和上一帧混合。变化小于 2 度直接忽略。罗盘旋转动画用了 CubicOut 缓动和 SpringOut 弹性回弹，看起来非常丝滑。指南针按 8 个方位推荐不同地区的美食——朝北推荐北方小吃，朝南推荐粤式美食，朝西南推荐川菜。

**EN**: The compass page uses the Compass API to read magnetometer data. I implemented exponential smoothing to reduce sensor noise — each new reading contributes only 12%, blended with the previous frame. Changes under 2 degrees are ignored. The rotation uses CubicOut easing with a SpringOut bounce-back for smooth, natural movement. The compass recommends dishes by direction — facing North gives Northern snacks, South for Cantonese, Southwest for Sichuan.

### 2.3 Camera / 相机 (≈ 40s)

**Show**: Switch to Album tab → Tap Camera → Take a photo → Photo appears in grid with category badge

**CN**: 相册使用 MediaPicker.CapturePhotoAsync 调用系统相机。拍照前检查并请求 CAMERA 权限。拍完照后弹出分类选择器——朋友聚会、好吃不贵、独自享受、深夜放毒、特别日子。照片保存在 App 本地目录，以 2 列网格展示，右下角有分类角标。

**EN**: The Album page uses MediaPicker.CapturePhotoAsync to invoke the system camera. Camera permission is requested and checked before capture. After taking a photo, a category picker appears — Friends & Fun, Tasty & Cheap, Solo Treat, Late Night Cravings, Special Days. Photos are saved to the app's local directory and displayed in a 2-column grid with category badges.

### 2.4 Geolocation / 定位 (≈ 30s)

**Show**: Open NearbyViewModel.cs in VS → Point to GPS + reverse geocoding code → Show error handling

**CN**: 定位功能使用 Geolocation API 获取 GPS 坐标，再通过 Geocoding API 进行反向地理编码。代码里处理了权限被拒绝、设备不支持、网络超时等多种异常情况，所有错误都有用户友好的提示。

**EN**: Geolocation uses the Geolocation API to obtain GPS coordinates, then Geocoding API for reverse geocoding. The code handles multiple error scenarios — permission denied, feature not supported, network timeout — all with user-friendly messages rather than raw exceptions.

### 2.5 HapticFeedback / 触觉反馈 (≈ 30s)

**Show**: Open RandomViewModel.cs → Point to HapticFeedback.Perform() calls → Explain try/catch wrapping

**CN**: 触觉反馈在模拟器中无法感受，但在真机上每次按钮点击都会产生短震，摇晃触发长震。请看代码——所有按钮和摇晃事件都调用了 HapticFeedback.Default.Perform。短震用 Click 类型，长震用 LongPress 类型。每个调用都包了 try/catch 确保设备不支持时也不会崩溃。

**EN**: Haptic feedback cannot be felt in the emulator, but on a physical device every button tap produces a short vibration and shaking triggers a longer one. Looking at the code — every button command and shake handler calls HapticFeedback.Default.Perform. Short taps use Click, shakes use LongPress. Each call is wrapped in try/catch so unsupported devices never crash.

**Show**: Complete flow walkthrough:

1. Discover tab → tap Pick Another → card animates
2. Tap Save → button turns grey "Saved"
3. Type "noodle" → tap Search → card shows noodle dish

11. Switch to Album → tap Gallery → pick from phone gallery
12. Tap category filter chips → photos filter by category

**CN**: 现在演示完整的功能流程。在 Discover 页面，点 Pick Another 随机换菜。点 Save 收藏，按钮变灰色显示 Saved。输入 noodle 点击 Search，从 53 道美食中搜索匹配。点 Share 调用系统分享面板。

切换到收藏页，顶部统计卡片显示收藏数量和最爱的分类。点击菜品写个人评价，保存后出现笔记角标。左滑删除有确认弹窗。点 Pick One for Me 从收藏随机选一个。底部 Clear All 清空所有收藏。

指南针页面转动手机，罗盘跟着旋转，实时推荐不同方位的美食。相册页面支持拍照和从相册导入，按分类筛选照片。

**EN**: Now I'll demonstrate the complete user flow. On Discover, tap Pick Another to randomly change dishes. Tap Save — the button turns grey showing Saved. Type noodle and hit Search to find a match from 53 dishes. Tap Share to invoke the system share sheet.

On Favourites, a stats card shows total saved dishes and your top category. Tap a dish to write a personal note — a badge appears when saved. Swipe left to delete with a confirmation dialog. Tap Pick One for Me to randomly select from your favourites. Clear All at the bottom empties everything.

The Compass page rotates with your phone, recommending region-specific dishes in real time. The Album page supports both camera capture and gallery import, with category filtering.

## 4. Validation & Error Handling (10%) / 验证与错误处理 (≈ 1.5 min)

**Show**: 
1. Leave search box empty → tap Search → Alert "Please enter a food name!"
2. Type "a" → tap Search → Alert "Name is too short! At least 2 characters."
3. Type "hotpot" → tap Search → Works normally
4. Open RandomViewModel.cs → show ValidateCraving() method
5. Scroll through code → show try/catch blocks everywhere

**CN**: 什么都不输入直接点 Search——弹出 Please enter a food name!。只输入一个字母——弹出 Name is too short! At least 2 characters.。输入正确内容——正常搜索。

验证逻辑在 ValidateCraving 方法里，返回元组 (isValid, message)。空值检查用 string.IsNullOrWhiteSpace，长度检查用 Length < 2。所有错误提示都是用户友好的自然语言。

每个硬件调用都包裹了 try/catch——指南针、相机、定位、触觉反馈、音效——任何一个功能失败都不会导致 App 崩溃。

**EN**: Leave the search box empty and tap Search — an alert says Please enter a food name!. Type just one letter — Name is too short! At least 2 characters. Enter valid text — works normally.

The validation logic is in ValidateCraving, returning a tuple (isValid, message). Empty check uses string.IsNullOrWhiteSpace, length uses Length < 2. All error messages are user-friendly natural language.

Every hardware call is wrapped in try/catch — compass, camera, geolocation, haptic, sound — so no single feature failure can crash the app.
